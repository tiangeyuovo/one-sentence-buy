# General Shopping Framework

本文件提供通用购物决策方法。

## 需求拆解

每个购物需求都拆成以下字段：

| 字段 | 说明 |
|---|---|
| product | 用户想买的商品或解决方案 |
| user | 谁使用 |
| scenario | 使用场景 |
| budget | 预算或价格偏好 |
| must_have | 硬性要求 |
| nice_to_have | 加分项 |
| avoid | 排除项 |
| platform | 购买平台 |
| urgency | 是否急用 |
| decision_goal | 省钱、性价比、一步到位、送礼、长期使用等 |

## 判断维度生成法

根据商品和使用场景，从下面维度里动态选择 4-7 个：

- 功能是否满足需求
- 是否适合真实使用场景
- 尺寸/规格/兼容性
- 材质和做工
- 安全性
- 耐用性
- 易用性
- 易清洁/维护
- 便携性
- 噪音/能耗/重量
- 售后和退换货
- 价格是否合理
- 店铺或品牌可信度
- 评价风险
- 外观风格
- 长期使用成本

## 商家话术识别

以下词不能直接当作事实，需要找证据支持：

- 爆款
- 网红
- 高端
- 专业
- 升级款
- 黑科技
- 旗舰同款
- 全网热卖
- 买一送多
- 进口品质
- 大牌平替

判断方法：

1. 看是否有可验证参数。
2. 看评价是否支持。
3. 看差评是否集中在核心功能。
4. 看多个商品是否都使用同一套夸张话术。
5. 看价格是否异常低但承诺过多。

## 差评权重

差评不是越多越差，关键看是否影响用户场景。

高权重差评：

- 核心功能失效
- 安全风险
- 尺寸/规格严重不准
- 做工影响使用
- 短期损坏
- 售后困难
- 与用户硬性要求冲突

中权重差评：

- 材质一般
- 有色差
- 体验不稳定
- 安装或使用麻烦
- 轻微噪音或味道

低权重差评：

- 物流慢
- 包装破损但商品无损
- 客服回复慢
- 个别主观审美不满意

## 推荐原则

- 硬性要求不满足，不能推荐为首选。
- 不推荐只因为便宜。
- 不推荐只因为销量高。
- 预算内优先选风险最低、场景最匹配、规格最清楚的。
- 如果多个候选都差不多，优先选退换货更方便、评价风险更少的。
